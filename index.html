# Password Manager (PWA) A private, offline password manager with a dark "Liquid Glass" iOS-inspired UI. All data is encrypted on-device with AES-256-GCM; there is no server, no account system, and no analytics. ## Running it Service workers require either `https://` or `http://localhost`, so opening `index.html` directly with `file://` will work for browsing the UI but the offline install prompt won't appear. To test the full PWA experience: ```bash cd password-manager python3 -m http.server 8080 # then open http://localhost:8080 in a browser ``` For real deployment, upload the whole folder as-is to any static host (GitHub Pages, Netlify, Vercel, S3, etc.) over HTTPS. On an iPhone, open the site in Safari, tap Share, then "Add to Home Screen" to install it as a native-feeling app. ## How the security model works - On first launch you create a **master password**. A random 16-byte salt is generated and PBKDF2-SHA256 (210,000 iterations) derives an AES-256 key from your password + that salt. - A small "verifier" blob is encrypted with that key and stored in IndexedDB, purely so the app can check a password attempt is correct without ever storing the password itself. - Every saved login is encrypted individually (unique IV per record) with AES-GCM before it touches IndexedDB. Nothing sensitive is ever written to `localStorage`. - Unlocking derives the key again in memory and decrypts entries for the session only. Locking the vault (manually, on inactivity, or when the tab is backgrounded) discards the key and the decrypted entries from memory. - There is intentionally no password-reset path — losing the master password means the encrypted data cannot be recovered. This is stated in the app itself (Settings → Forgot Password, and during first-time setup). ## Structure ``` index.html All screens (setup, unlock, vault, add/edit, details, categories, settings) plus modal sheets css/style.css Liquid Glass design system js/crypto.js PBKDF2 key derivation + AES-GCM encrypt/decrypt, password generator, strength scoring js/db.js IndexedDB wrapper (vault meta + entries stores) js/app.js App state, screen/sheet management, all UI wiring manifest.json PWA manifest (icons, theme, install metadata) sw.js Service worker (offline app-shell caching) icons/ App icon at all required PWA sizes, incl. maskable ```
